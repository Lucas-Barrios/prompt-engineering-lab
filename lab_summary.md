# Lab Summary

I tested three broken chatbot prompts by running them over and over to see where they failed. The first versions were pretty bad—only 10-30% consistent because I didn't give the model any structure or rules. Then I tried adding clear instructions like "respond with ONLY one word" and defining what format I wanted, which fixed sentiment and data extraction completely but product descriptions were still all over the place. The breakthrough came when I added examples showing exactly what good output looks like and set temperature to 0 so it would stop being creative. That got everything to 95-100% consistency. The biggest lessons were that being super explicit about format (like "return ONLY JSON, no explanation") and showing examples make a huge difference. Looking back, I wasted time running 15 tests on the broken prompts when the problems were obvious after 5—I should've moved to fixing them faster.
